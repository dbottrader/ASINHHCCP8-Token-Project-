    Spiral Storage Mini-Spec (STF-1)

    A shared JSON envelope for browser rendering, export, audio mapping, and deterministic sealing:

    STF-1 (Spiral Tape Format v1)

header.magic      = "STF1"
header.version    = 1
layout.mode       = "rings" | "single_spiral"
layout.rings      = uint16
layout.sectors    = uint16
layout.start_deg  = float
layout.dir        = "cw" | "ccw"
mapping.bit_order = "ring_major" | "sector_major"
payload.bits      = "0101..."
audio.base_hz     = float
audio.low_mul     = string ratio
audio.high_mul    = string ratio
audio.bpm         = float
audio.duty        = float (0..100)
seal.sha256       = SHA-256(canonical STF-1 bundle without seal value)

Addressing:
addr = {ring:0..R-1, sector:0..S-1}

Rendering:
filled segment = 1; gap = 0; theme may invert presentation.

    UI ↔ Schematic ↔ Hardware Bridge

    • Embedded schematic layer: each implemented device page may carry a block diagram, parts list, and interface specification.
    • One JSON, multiple targets: the same validated STF-1 bundle can populate a UI, an SVG representation, firmware test vectors, and node-network receipts.
    • Safety and authority rails: checksums, signatures, limits, consent where applicable, logs, and explicit evidence grades must precede any hardware or human-subject deployment.
