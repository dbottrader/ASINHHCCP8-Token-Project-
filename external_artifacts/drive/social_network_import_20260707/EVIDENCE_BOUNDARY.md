# Evidence Boundary — ASINHHCCP8 Social Network / Token Layer

## Valid Public Framing

This repository may be described as an early public prototype exploring:

```text
human + AI agent collaboration
    -> verifiable interaction receipts
    -> governance / moderation checks
    -> contribution accounting
    -> optional internal reward credit
```

## Valid Current Claim

```text
ASINHHCCP8 / ASIN-NCEA token-layer scaffold for testing replayable receipts, Merkle commitments, runtime accountability, and internal contribution credit for human + AI collaboration workflows.
```

## Non-Claims

This repository does **not** currently claim:

- production-certified cryptography;
- externally traded token issuance;
- audited financial settlement;
- legal/financial/investment authority;
- externally deployed social network infrastructure;
- proof that all Drive artifacts are production-ready;
- scientific proof of physical frequency effects.

## Evidence Ladder

| Tier | Meaning in this repo |
|---|---|
| E0 | idea / sketch / prompt / symbolic note |
| E1 | Drive artifact, document, code draft, image, or imported file |
| E2 | local run with command, output, and receipt |
| E3 | reproducible run from clean instructions / CI |
| E4 | external reviewer or independent reproduction |
| E5 | deployed production controls with monitoring/accountability |

## Promotion Rule

No artifact should be promoted higher than its weakest unsupported dependency.

## Social Network Rule

Reward must follow verified contribution, not posting volume.

The project should distinguish:

```text
post/noise            -> no reward by default
review/build/help     -> possible contribution event
verified receipt      -> internal credit eligibility
replayable evidence   -> stronger claim
```

## Collaboration Rule

Useful reviewers should be asked to inspect, run, break, and improve the structure rather than accept claims by reputation.
